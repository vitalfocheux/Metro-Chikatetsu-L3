TP9 - Test fonctionnel

Objectifs du TP :

  • Réaliser des tests fonctionnels de l'application.

Description de l'application à tester

La description complète est disponible ici. Le modèle de données considéré est rappellé ci-dessous :

Dans ce TP, nous allons n'allons pas écrire de nouvelle classe, le système étant désormais complet.

Nous allons nous focaliser sur l'écriture de tests fonctionnels qui ont pour objectif de réaliser des scénarios d'utilisation classique de l'application.

Travail à réaliser

Ecrivez vos tests fonctionnels dans une classe nommée TestsFonctionnels.java. Cette classe sera hors du package manipulé jusqu'à présent.

Le principe de fonctionnement de l'application sera le suivant :

  • Le réseau est instancié via un fichier décrivant sa structure.
  • Le client utilise la TicketMachine pour instancier son ticket.
  • Le ticket est passé dans une barrière d'entrée, situé dans une certaine station du réseau.
  • Le client ressort à une autre station, en empruntant la barrière de sortie.
  • Si le client n'avait pas provisionné assez d'argent sur son ticket, il peut l'ajuster en vue de sortir.

Vous pouvez travailler de façon indépendante au sein d'un groupe (chacun écrit sa propre classe, et suffixe le nom de celle-ci avec ses initiales) ou faire un peu de pair-programming (ce qui s'apparent plus à du pair-testing pour le coup).

Les tests fonctionnels devront être réalisés pour la prochaine séance de TP.
